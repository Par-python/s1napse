# 50 Hz Telemetry Sampling Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Raise the telemetry sampler from ~22 Hz to ~50 Hz and switch lap exports/imports to gzip-compressed `.json.gz` files.

**Architecture:** No structural changes. The sampler thread, render timer (5 Hz), and animation timer (60 Hz) are already decoupled — only the sampler's sleep interval and the lap I/O code paths change. Existing plain-`.json` lap files will not be readable after this change (per spec: clean break, no backward-compat).

**Tech Stack:** Python 3, PyQt6, `gzip` (stdlib), `json` (stdlib).

**Spec:** [docs/superpowers/specs/2026-04-24-50hz-sampling-design.md](../specs/2026-04-24-50hz-sampling-design.md)

**Note on testing:** This repo has no pytest infrastructure (only `acc_test.py`, a live-ACC manual script). Verification is structural (importability, format round-trip via `python -c`) plus a final manual smoke test in ACC. Adding pytest scaffolding for a sleep-value change and a `json.dump`→`gzip.open` swap would be ceremony, not value.

```bash
import gzip, json, tempfile, os
payload = {'lap_number': 1, 'data': {'speed': [100.0, 110.5, 120.2]}}
fd, path = tempfile.mkstemp(suffix='.json.gz')
os.close(fd)
with gzip.open(path, 'wt', encoding='utf-8', compresslevel=6) as f:
    json.dump(payload, f)
with gzip.open(path, 'rt', encoding='utf-8') as f:
    loaded = json.load(f)
assert loaded == payload, f'round-trip mismatch: {loaded}'
os.remove(path)
print('round-trip OK')
"
```
Expected: `round-trip OK`.
